几个问题

1、JavaScript中的变量在内存中的具体存储形式

变量的存储形式为堆和栈
开发人员不能直接操作堆区，堆区数据由JS引擎操作完成
基本数据类型：栈
引用对象类型：堆

分析：JS预加载阶段，JS引擎只是在内存的栈区为每个变量分配了内存，指定了标示符，并未为其指定值。

obj2与obj实际上已经指向了同一个堆区的对象，然后obj2修改了这个对象的某个属性值。然后函数执行完毕，obj2这个局部变量没有引用将会被GC回收。

3、精度丢失问题，如何避免？

计算机的二进制实现和位数限制有些数无法有限表示。就像一些无理数不能有限表示，如 圆周率 3.1415926…，1.3333… 等。JS 遵循 IEEE 754 规范，采用双精度存储（double precision），占用 64 bit。

  • 1位用来表示符号位
  • 11位用来表示指数
  • 52位表示尾数

举个栗子：0.1 >> 0.0001 1001 1001 1001…（1001无限循环）
此时只能模仿十进制进行四舍五入了，但是二进制只有 0 和 1 两个，于是变为 0 舍 1 入。这即是计算机中部分浮点数运算时出现误差，丢失精度的根本原因。
JS 中能精准表示的最大整数是 Math.pow(2, 53)，十进制即 9007199254740992。
大于 9007199254740992 的可能会丢失精度

解决方式：
对于小数：把小数放到位整数（乘倍数），再缩小回原来倍数（除倍数）
(0.1*10 + 0.2*10) / 10 == 0.3 // true
